01 / Intro
Workshop-Demo · 6. Mai 2026

Sechs Artikel.
30 Minuten.
Vollautomatisch.

Wie ein einzelnes Python-Script alle fünf Minuten einen recherchierten Fachartikel inklusive Bild generiert, in den Blog einbettet und auf GitHub Pages live nimmt — ohne menschliche Interaktion zwischen den Iterationen.

02 / Was passiert

Der Auftrag

Eine halbe Stunde lang, alle fünf Minuten, ein neuer Artikel auf dem Blog — mit Hero-Bild, ein Artikel zusätzlich mit Video-Embed.

00:00Predictive Maintenance + Video
05:00LoRaWAN Submetering
10:00ISO 50001 2026
15:00CO₂-Bilanz/m²
20:00Smart Meter Österreich
25:00Mieterstrom digital

Sechs Topics, sechs Tick-Zeitpunkte, sechs git pushes. Der gesamte Lauf endet nach 30 Minuten — und der Prozess hält sich selbst an.

03 / Architektur

Was ein Tick tut

1 · TopicAus statischer Liste den nächsten Topic-Eintrag (Slug, Titel, Keyword, Tag) ziehen.
2 · TextOpenRouter → Gemini 2.5 Flash Lite. Prompt erzwingt JSON mit TLDR, Sections, FAQ.
3 · BildOpenRouter → Gemini 3.1 Flash Image. Prompt mit konstanter Stil-DNA + Topic-Motiv.
4 · RenderPython-Templating in HTML mit Schema.org JSON-LD (Article, BreadcrumbList, FAQPage).
5 · IndexKarte zu index.html hinzufügen, neuer Eintrag in sitemap.xml.
6 · Pushgit add . && git commit && git push — GitHub Pages baut automatisch.

Eine Iteration dauert in der Praxis 15–35 Sekunden — der Rest des 5-Minuten-Fensters ist Idle-Time.

04 / Modelle

Warum diese Modelle

Text · google/gemini-2.5-flash-lite

Sehr günstig (~$0.10 / $0.40 pro 1M Token In/Out), beherrscht JSON-Mode, deutsche Fachsprache solide. Sechs Artikel kosten zusammen unter 1 Cent.

Bild · google/gemini-3.1-flash-image-preview

Nano-Banana-Nachfolger. Fotorealistisch, akzeptiert Stil-DNA-Prompts mit „no people, no text, no logos", liefert in 10–25 Sekunden. Ca. 4 Cent pro Bild.

Provider-Routing über OpenRouter: ein API-Key, beide Modelle. Falls ein Modell Limits erreicht, wechselt OpenRouter transparent auf einen Backup-Provider — kein Code-Change nötig.

Gesamtkosten der 30-Minuten-Demo: ca. 0,25 € Modellkosten, plus statisches GitHub-Pages-Hosting (kostenlos).

05 / Code

Die Schleife — minimal

for i, topic in enumerate(TOPICS):
    target = start + i * INTERVAL          # Tick-Zeit
    time.sleep(max(0, target - time.time()))
    try:
        gen_one(topic)                     # Text + Bild + HTML + Push
    except Exception as e:
        log(f"FAILED {topic['slug']}: {e}") # Fehler tolerieren, weiter

Drei Eigenschaften, die das hier robust machen:

  • Absolute Zeitstempel statt sleep(300) — verhindert Drift, wenn ein Lauf länger dauert als geplant.
  • Try/Except pro Topic — ein einzelner API-Fehler kippt nicht die gesamte Pipeline.
  • Idempotenter HTML-Output — gleicher Slug überschreibt sauber, Sitemap-Eintrag wird nur einmal angehängt.
06 / Qualität

Was nicht automatisierbar ist

Die ehrliche Seite: ein 30-Sekunden-Modell-Lauf ersetzt keine Fachredaktion. Drei Leitplanken halten die Qualität in einem vertretbaren Bereich.

E-E-A-T

Demo-Hinweis auf jeder Seite, klare Auszeichnung als KI-generiert, Autorenangabe + Kontakt — keine Täuschung.

Keine Fake-Studien

Prompt verbietet erfundene Quotes und konkrete Studienzahlen. Quellenhinweise nur generisch.

Bandbreiten statt Punkte

Erzwungene „15–22 %" statt „17,3 %" — verhindert scheingenaue Aussagen, die nach Fakten klingen, aber Modell-Halluzination sind.

Für ein produktives Setup würde eine zweite Stufe folgen: redaktionelle Freigabe via Pull Request statt direktem Push auf main.

07 / Mehr

Wie geht es weiter?

Der Bauplan eignet sich für jede Art Content-Pipeline: News-Briefing, interne Reports, Wissensbasis-Updates, Status-Pages.

  • Statt Topic-Liste: RSS-Feed oder Issue-Tracker als Quelle.
  • Statt direkter Push: Pull Request, mit menschlichem Review-Step.
  • Statt Cron im Bash: GitHub Actions oder ein Scheduler im Hosting-Stack — keine eigene Infrastruktur.
  • Feedback-Loop: Klicks/Lesezeit pro Artikel zurück in die Topic-Auswahl der nächsten Iteration.
